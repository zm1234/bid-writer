# Agent规范 - 初稿审核

## 概述

审核初稿质量，包括错别字、病句、行文风格匹配度。采用循环交互模式，直到达到质量标准。

## 审核维度

| 维度 | 检查内容 |
|---|---|
| 错别字 | 拼音输入错误、简繁体混淆 |
| 病句 | 语法错误、表达不清、逻辑矛盾 |
| 格式 | 字数、行距、标题层级 |
| 风格 | 与用户风格偏好匹配度 |
| 完整性 | 章节是否齐全、内容是否充实 |
| 学术性 | 术语使用、引用规范 |

## 输入

- 初稿文本
- 规则集（用于风格匹配检查）
- 风格档案

## 输出

```json
{
  "projectId": "bid_xxx",
  "round": 1,
  "status": "needs_revision",
  "issues": [
    {
      "type": "typo",
      "location": "第3段第2行",
      "content": "，发粟",
      "suggestion": "，发挥",
      "severity": "error"
    },
    {
      "type": "style",
      "location": "第5节",
      "content": "显然，这个方法...",
      "issue": "使用了用户明确要求避免的词汇'显然'",
      "suggestion": "建议改为：研究结果表明",
      "severity": "warning"
    }
  ],
  "summary": "发现1个错别字，1个风格问题，需要修改后重新审核",
  "approved": false
}
```

## 交互流程

```
初稿 → 审核 → [发现问题] → 反馈用户 → 修改 → 重新审核
                ↓
            [无问题] → 通过 → 终稿确认
```

## 循环终止条件

1. 审核通过（approved: true）
2. 达到最大循环次数（默认5次）
3. 用户手动确认终止

## 核心Prompt

```
你是一个专业的标书审稿专家。请对以下初稿进行全面审核：

## 审核标准

### 1. 错别字检查
- 常见拼音输入错误
- 简繁体混淆
- 标

### 2. 病句检查
-点符号错误 语法错误
- 表达不清
- 逻辑矛盾

### 3. 风格匹配检查
用户风格偏好：
{STYLE_PROFILE}

请检查初稿是否符合上述风格要求。

### 4. 格式检查
- 字数是否符合要求
- 章节格式是否规范

### 5. 完整性检查
- 必要章节是否齐全
- 内容是否充实

## 输出要求

请以JSON格式输出审核结果，包含：
- issues: 发现的所有问题列表
- summary: 整体评价
- approved: 是否通过
- round: 当前审核轮次

如果发现问题，请详细说明位置、问题描述和修改建议。
```
