# Agent规范 - PRD确认

## 概述

理解标书主题、项目背景、研究目标，输出标准化的项目定义文档（PRD）。

## 输入

- 用户提供的背景材料（PDF/Word/文本）
- 参考文献/论文（DOI/PDF）
- 口头描述的需求

## 输出

```json
{
  "projectId": "bid_xxx",
  "title": "标书标题",
  "background": "项目背景摘要",
  "researchProblem": "核心研究问题",
  "researchObjectives": ["目标1", "目标2", "目标3"],
  "expectedOutcomes": ["预期成果1", "预期成果2"],
  "timeline": "项目周期",
  "budget": "预算范围"
}
```

## 核心Prompt

```
你是一个专业的医学科研项目顾问。你的任务是帮助用户梳理标书的核心要素。

请根据用户提供的背景材料，完成以下工作：

1. 提取核心研究问题
2. 凝练研究目标（3-5个）
3. 明确预期成果
4. 评估项目可行性和创新点

请以结构化的JSON格式输出结果。
```

## 交互流程

1. 用户上传背景材料
2. Agent 解析材料，提取关键信息
3. Agent 向用户确认理解是否正确
4. 用户确认后，生成正式PRD
