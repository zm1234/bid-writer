# ADR-0001: 多Agent协作架构

- Date: 2026-03-08
- Status: Proposed
- Deciders: zhoumiao
- Related: ResearchX ADR-0001

## Context

标书撰写是一个复杂的线性+迭代过程，涉及：
1. PRD确认（理解项目背景）
2. 规则规划（格式/风格/硬性要求）
3. 初稿撰写（生成内容）
4. 初稿审核（多轮交互修改）
5. 终稿确认（输出定稿+阅读指南）

需要设计一套多Agent协作机制。

## Decision

采用 **串行+循环** 混合架构：

```
PRD确认 → 规则规划 → 初稿撰写 → [初稿审核]ⁿ → 终稿确认
                              ↑
                              └─ 循环交互直到审核通过
```

每个Agent职责单一，通过共享上下文（项目ID）传递数据。

## Agent职责边界

| Agent | 输入 | 输出 | 状态 |
|---|---|---|---|
| PRD确认 | 背景材料、论文 | 项目定义 | 一次性 |
| 规则规划 | 用户风格偏好、历史 | 规则集+风格档案 | 一次性+可更新 |
| 初稿撰写 | 规则集+项目定义 | 完整初稿 | 一次性 |
| 初稿审核 | 初稿 | 审核意见 | 可循环 |
| 终稿确认 | 审核后定稿 | 终稿+阅读指南 | 一次性 |

## 数据流设计

- **项目上下文**：每个标书项目有唯一ID，存储所有中间产物
- **风格历史**：SQLite数据库，记录用户偏好、常用表达、拒用词汇
- **经验库**：每次撰写后的反馈总结，用于Agent自进化

## Consequences

- 正面：流程清晰、易调试、每个Agent可独立优化
- 负面：初期需要较多协调工作

## Implementation Notes

- 参考 ResearchX 的 pi-mono 作为 Agent 运行时
- 使用 Next.js Route Handlers 作为 Agent API
- 风格历史用 SQLite 存储（参考 ResearchX 数据层）
